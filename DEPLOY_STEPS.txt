====================================================================
SIMPLE DEPLOYMENT STEPS - SEND BACKEND TO FRONTEND DEVELOPER
====================================================================

IMPORTANT: Don't use Netlify! It doesn't support Express.js + Socket.io
Use Railway instead (easiest + free tier)

--------------------------------------------------------------------
STEP 1: PUSH CODE TO GITHUB
--------------------------------------------------------------------

1. Create new GitHub repository:
   - Go to github.com
   - Click "New repository"
   - Name: handyman-backend
   - Public or Private
   - Don't initialize with README (you already have one)

2. Push your code:
   cd c:\Users\Tommy\Desktop\handyman
   git init
   git add .
   git commit -m "Backend ready for deployment"
   git branch -M main
   git remote add origin https://github.com/YOUR_USERNAME/handyman-backend.git
   git push -u origin main

--------------------------------------------------------------------
STEP 2: DEPLOY TO RAILWAY (5 MINUTES)
--------------------------------------------------------------------

1. Go to: https://railway.app

2. Click "Start a New Project"

3. Click "Deploy from GitHub repo"

4. Sign in with GitHub and authorize Railway

5. Select "handyman-backend" repository

6. Railway will automatically:
   - Detect it's a Node.js app
   - Run npm install
   - Start the server
   - Give you a URL!

--------------------------------------------------------------------
STEP 3: ADD ENVIRONMENT VARIABLES IN RAILWAY
--------------------------------------------------------------------

1. In Railway dashboard, click your project

2. Go to "Variables" tab

3. Click "New Variable" and add these (IMPORTANT!):

   NODE_ENV = production
   PORT = 5000
   
   SUPABASE_URL = https://your-project.supabase.co
   SUPABASE_ANON_KEY = your_anon_key
   SUPABASE_SERVICE_KEY = your_service_key
   
   JWT_SECRET = generate_a_random_32_character_string
   
   CORS_ORIGIN = *
   
   (Add others from .env.example as needed)

4. Click "Deploy" or wait for auto-redeploy

--------------------------------------------------------------------
STEP 4: GET YOUR API URL
--------------------------------------------------------------------

1. In Railway, go to "Settings" tab

2. Under "Domains", you'll see:
   https://handyman-backend-production.up.railway.app

3. COPY THIS URL - this is what you send to frontend dev!

--------------------------------------------------------------------
STEP 5: TEST YOUR DEPLOYED BACKEND
--------------------------------------------------------------------

Open browser or use curl:

https://YOUR-RAILWAY-URL.com/health

Should return:
{
  "status": "ok",
  "timestamp": "...",
  "uptime": 123,
  "environment": "production"
}

If you see this, YOUR BACKEND IS LIVE! ✅

--------------------------------------------------------------------
STEP 6: SEND INFO TO FRONTEND DEVELOPER
--------------------------------------------------------------------

Send these 3 files:
1. API_FOR_FRONTEND.md - API documentation
2. The Railway URL - https://your-app.up.railway.app
3. Test credentials (create test accounts first)

Tell them:
- API Base URL: https://your-app.up.railway.app/api
- Socket.io URL: wss://your-app.up.railway.app
- Read API_FOR_FRONTEND.md for all endpoints

--------------------------------------------------------------------
